6. Risk assessment

The risk assessment methodology is selected in from the following list:

  • Highest risk: Maximum risk for the category/indicator/sub-indicator triple.
  • Average risk: Average risk for the category/indicator/sub-indicator triple
  • Weighted average. A weighted average is calculated, enabling the super administrator to assign different weights to the different types of ESG Assessment reports.

In the case where weighted average is selected, a further screen is provided for the user to enter weights for the different input types that were registered during system configuration. The list is dynamically created based on the following:

  • Geography is added if a geographic risk metric is uploaded in the first tab of the template spreadsheet.
  • Product risk listings is added if a geographic risk metric is uploaded in the first tab of the template spreadsheet.
  • If any producer role has been assigned the ability to submit an assessment, SAQs will be listed.
  • If any administrator role has been assigned the ability to submit DNA test results, it will appear in the list.

Each ESG Assessment role that has been assigned the ability to submit reports will appear separately in the list. This allows the super administrator to assign more weight to reports from one assessor than others.

Regardless of the risk methodology selected, the Indicator List and ESG Assessment reports are combined together as:

Weight = Risk of reported case (low=1; medium=2; high=3) X Severity from Indicator List

This means that each report will result in a score between 1 to 15 (1 indicating low risk, 15 indicating extreme risk), signifying the severity of the reported incidence.

Once a complete list of weights for each report is calculated, they are combined using the selected methodology.
