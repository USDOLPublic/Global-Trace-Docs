Appendix E. Open Supply ID

Included below is the flow chart how to register a new facility OS ID

As the diagram shows, when registering a new facility there are 3 status responses:

  • MATCHED: It means the 3rd party algorithm indicates that with the information we populated there is an oar id that matched with this information so that we can choose this one, if not we will have the 2 cases.
  • NEW-FACILITY: It means there are no oar ids in the 3rd party database that match with the information we populated to register the, so that the 3rd party will create the new one and then we can take it to assign to our facility. If the status is not NEW-FACILITY it can be the 3rd case
  • POTENTIAL-MATCH: It means the 3rd party algorithm checked there are some oar ids that are similar with some part of information we populated. It will show all these oar ids so that we can confirm which one is matched with our facility. If all of them are not the one we need, we can reject and then the 3rd party will register the new one with our facility information.
  • Sometimes, the 3rd party will response the ERROR-MATCHING status it indicates that there is a error with the 3rd party system of we can not register new oar id with our facility information
