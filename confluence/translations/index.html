Appendix B. Translations

For Global Trace to support complete user interface internationalisation, there are two types of internationalisation that are required: system translation; and content translation.

System Translation

The system can be translated by copying all fines in the following directory: src/i18n/en

Create a new folder, using the language code for the new language. Translate all phrases listed in the JSON files. Perform a similar process for all files in the following directory: i18n/en

When a system translation has been added, the user will be able to select to use the translation from the Global Trace page header.

Content translation

Next, all items specified in steps 1,2,4,5 must be able to be translated.

Translations of products, roles, indicator lists, and assessments are uploaded in a similar manner across the platform. First, once the item has been configured, a template is automatically generated by the system and is available for download. The template will include one line for each language that the system has an interface defined in. See box below.
