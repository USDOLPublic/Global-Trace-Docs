Global Trace

1. Product definition

For each input or output product that will be traced in Global Trace, the super administrator must specify the key data elements (KDEs) or product attributes that will be required to be collected.

Attribute definition

An attribute is created by the super admin and has a name, type, category, and other optional defined values.

  1. An attribute type can be one of three types: PRODUCT_ID, PRODUCT_QUANTITY, and OTHER.
  • PRODUCT_ID can be used to identify a unique product, i.e. Personal ID for a person, Serial Number for an mobile device, Code for cotton,
  • PRODUCT_QUANTITY is the quantity that the product is bought or sold.
  1. An attribute category is one of eight types: TEXT, NUMBER, PERCENTAGE, DATE, LIST, NUMBER_PAIR_UNIT, COUNTRY_PROVINCE_DISTRICT, ATTACHMENTS.
  • PERCENTAGE defines that the attribute must be a number from 0 to 100.
  • LIST defines that this attribute is a list of values, such as attribute Color with three options: black, pink, and red.
  • NUMBER_PAIR_UNIT defines an attribute of two connected properties, for example, attribute Price with type of NUMBER and currency is one of PKR, USD, and EUR.
  • COUNTRY_PROVINCE_DISTRICT is the address containing three values: Country ID, Province ID, and District ID in one attribute.
  • ATTACHMENTS is used to contain uploaded certifications or proofs of the product. This category must contain the blobName and fileName of the uploaded file.

Note

An attribute of type PRODUCT_ID must have category TEXT. An attribute of type PRODUCT_QUANTITY must have category NUMBER_UNIT_PAIR.
